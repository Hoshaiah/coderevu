## Context

`app/models/ticket.rb` uses ActiveRecord's `enum` to track ticket status. The model also has a hand-written class method called `active` that was added years ago and returns tickets assigned to the current sprint with additional filters (not-closed, assigned, due within 30 days). This `active` method is called from eight controllers and two background jobs.

After Rails was upgraded from 6.0 to 7.1, an engineer added `enum :status, { open: 0, active: 1, closed: 2 }` to replace a set of integer constants. CI passed because the `active` named scope generated by `enum` happens to return a non-empty relation for most test fixtures.

In production, the sprint board shows the wrong tickets — it no longer filters by assignment or due date, and closed/unassigned tickets appear on the board. No exception is raised.

## Buggy code

```ruby
class Ticket < ApplicationRecord
  belongs_to :assignee, class_name: "User", optional: true
  belongs_to :sprint, optional: true

  enum :status, { open: 0, active: 1, closed: 2 }

  # Returns tickets that should appear on the sprint board.
  def self.active
    where("due_at <= ?", 30.days.from_now)
      .where.not(assignee_id: nil)
      .where.not(status: :closed)
  end

  def self.overdue
    where("due_at < ?", Time.current).active
  end
end
```
