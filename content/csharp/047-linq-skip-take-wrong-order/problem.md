## Context

This method lives in `Repositories/ProductRepository.cs` in an e-commerce API. It is called by the `GET /products` endpoint which accepts `page` and `pageSize` query parameters. Products are ordered by creation date and the default page size is 20. The repository targets SQL Server via Entity Framework Core.

Customers report that the first page of results is always empty, but page 2 and beyond work fine. Support tickets note the problem started after a recent refactor that extracted pagination logic into a shared helper. Load tests show the query executes without error and returns HTTP 200 with an empty `items` array for `page=1`.

The developer who wrote the refactor double-checked the SQL generated by EF and noted it contains both `OFFSET` and `FETCH NEXT` clauses, which looks correct at a glance.

## Buggy code

```csharp
public class ProductRepository
{
    private readonly ShopDbContext _db;

    public ProductRepository(ShopDbContext db)
    {
        _db = db;
    }

    public async Task<List<Product>> GetPageAsync(int page, int pageSize)
    {
        // page is 1-based
        return await _db.Products
            .OrderBy(p => p.CreatedAt)
            .Take(pageSize)
            .Skip((page - 1) * pageSize)
            .ToListAsync();
    }
}
```
